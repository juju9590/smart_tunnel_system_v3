# 🚇 AI Smart Tunnel Monitoring System

> **CCTV 기반 차량 흐름 분석, 사고 위험 탐지, 차선 상태 추정, 센서리스 환기 대응을 통합한 AI 기반 스마트 터널 관제 시스템**

본 프로젝트는 터널 CCTV 영상을 기반으로 차량 객체탐지, 추적, ROI 자동설정, 차선추정, 교통상태 분석, 사고탐지, 환기 대응 지원을 수행하는 **실시간 스마트 터널 관제 시스템**입니다.

단순한 객체탐지 데모가 아니라, 실제 관제 환경을 가정하여 **AI 분석 파이프라인 + Flask API + React 대시보드 + 성능평가 체계**까지 직접 구현한 포트폴리오 프로젝트입니다.

---

## 📌 Project Overview

터널은 폐쇄적인 공간 특성상 사고 발생 시 대응 지연이 큰 위험으로 이어질 수 있습니다.  
기존 관제 방식은 CCTV를 사람이 직접 확인하거나, 별도의 센서에 의존하는 경우가 많아 다음과 같은 한계가 있습니다.

- 사고 인지 지연
- 정체/혼잡 상태 판단의 어려움
- 터널별 센서 설치 비용 부담
- 환기 대응 판단 기준의 제한
- 운영자 의존적인 관제 방식

본 프로젝트는 이러한 문제를 해결하기 위해 **CCTV 영상만으로 터널 내 교통 흐름과 위험 상황을 분석하는 AI 기반 관제 시스템**을 구현했습니다.

  ↓

## 🖥️ Main Features

### 1. YOLO 기반 차량 객체탐지

터널 CCTV 영상에 특화된 차량 탐지를 위해 YOLO 모델을 학습하고, 실시간 처리 속도를 고려하여 경량 모델을 적용했습니다.

- 터널 CCTV 영상 내 차량 객체 탐지
- 터널 전용 데이터셋 구성 및 학습
- Precision / Recall / mAP50 기반 성능평가
- 실시간 처리를 위한 FPS 검증

---

### 2. ByteTrack 기반 차량 추적

단일 프레임 탐지만으로는 차량 흐름을 분석하기 어렵기 때문에, ByteTrack을 활용하여 차량 ID를 추적했습니다.

- 차량별 ID 추적
- 프레임 간 이동량 계산
- 차량별 이동 궤적 관리
- 평균속도 및 체류시간 분석 기반 제공
- 교통상태 판단과 사고탐지 로직의 기초 데이터로 활용

---

### 3. ROI 자동설정

터널 CCTV는 카메라 각도와 도로 구조가 다르기 때문에, 고정된 분석 영역을 사용하기 어렵습니다.  
이를 해결하기 위해 차량 위치 데이터를 기반으로 터널 내 주요 분석 영역을 자동 설정했습니다.

- 차량 bbox 하단 좌표 기반 ROI 후보 수집
- 일정 프레임 누적 기반 ROI 안정화
- 터널 구조에 맞춘 분석 범위 자동 설정
- 분석 대상 차량과 배경/외부 영역 분리
- ROI Match Rate 기반 성능 평가

---

### 4. 차선추정

사고탐지와 교통상태 판단에는 차량이 어느 차선에서 움직이는지 파악하는 것이 중요합니다.  
차량 이동 위치와 분포를 기반으로 차선 구조를 추정하고, 조건 충족 후 차선 템플릿을 확정하는 방식으로 구현했습니다.

- 차량 위치 기반 차선 후보 추정
- 차선 템플릿 생성 및 저장
- 운영자 수동 차선 수 설정 지원
- CCTV별 차선 메모리 재사용
- 조건 충족 후 차선추정 성공률 100% 달성

---

### 5. 교통상태 분석

차량별 이동량을 기반으로 터널 내 교통 흐름을 정상 / 혼잡 / 정체 상태로 분류했습니다.

단순히 한 순간의 속도만 보는 것이 아니라, 일정 프레임 동안 누적된 평균 흐름을 사용하여 화면 흔들림이나 일시적인 가림에도 안정적으로 상태를 판단하도록 설계했습니다.

- 차량별 이동량 기반 평균속도 계산
- 300프레임 누적 평균 기반 상태 판단
- 정상 / 혼잡 / 정체 상태 분류
- 일시적인 흔들림과 가림 현상에 대한 상태 안정화
- 웹 대시보드에 현재 교통상태와 평균속도 시각화

---

### 6. 사고탐지

사고탐지는 단순히 차량이 멈춘 상태만으로 판단하지 않고, 사고 영상에서 나타나는 패턴을 기반으로 설계했습니다.

주요 감지 대상은 다음과 같습니다.

- 후방추돌
- 비정상 정지
- 흐름 단절
- 사고 이후 차량 정체 패턴
- 특정 구간 고정 장애물성 패턴

혼잡/정체 상황에서의 오탐을 줄이기 위해 여러 방어 조건을 추가했습니다.

- queue-only 상황 방어
- 작은 원거리 객체 쌍 방어
- 단순 정체와 사고 패턴 분리
- bbox split 기반 false stop 방어
- 사고 확정 후 popup / event log 처리

---

### 7. 센서리스 환기 대응 지원

터널 내 공기질 센서가 없다는 가정에서, 영상 기반 교통 흐름 정보를 활용하여 환기 대응 지표를 설계했습니다.

차량 흐름이 느려지고 체류시간이 증가하면 터널 내부 공기질 악화 가능성이 커진다고 보고, 현재 상태와 5분 후 예상 상태를 분리하여 위험도를 계산했습니다.

- 차량 수 기반 혼잡도 반영
- 평균속도 기반 흐름 저하 반영
- 교통밀도 기반 공기질 위험도 산출
- 평균체류시간 기반 위험 점수 계산
- 현재 공기질 위험도와 5분 후 예상 위험도 분리 표시
- 대시보드에서 환기 대응 상태 시각화
